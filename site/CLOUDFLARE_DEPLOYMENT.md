# Cloudflare Pages — marketing site

This folder is the public marketing site for **lokki.starstucklab.com**.

The fleet-management dashboard at `web/app/` is unrelated — it ships
inside the firmware (flashed to the coordinator) and is also served
locally on each device's web server. It is **not** publicly deployed.

## Cloudflare Pages settings

In the existing Cloudflare Pages project for `lokki.starstucklab.com`,
update **Settings → Builds & deployments → Build configurations**:

| Field | Value |
|---|---|
| Framework preset | None (static HTML) |
| Build command | _(leave empty)_ |
| **Build output directory** | `site` |
| Root directory | `/` (repository root) |

The previous value was `web/app`. Change it to `site` and trigger a new deployment.

No environment variables are required. The site is fully static — HTML,
CSS, one tiny script, and the WebP assets generated by
[`utils/marketing/`](../utils/marketing/README.md).

## Local preview

There's no build step. Open `site/index.html` directly in a browser, or
serve the folder:

```bash
cd site && python3 -m http.server 4000
# → http://localhost:4000
```

## Configuration files in this folder

- `_headers` — security and cache headers for Cloudflare Pages
- `_redirects` — currently empty; reserved if URL rewrites are added later
