Lokki Configuration Tools

Static tools for configuring and setting up Lokki LED lighting controllers. Runs entirely in your browser — nothing is uploaded.

⚙️
Config Builder
Build a config.json for any Lokki unit — coordinator or leaf. Configure LED schedules, PIR motion responses, relay control, scenes, and LoRa network settings. Includes an integrated sun times generator for solar-based schedules.

Setup Workflow

Before you start — you'll need:
  • MicroPython UF2 for Pico 2 W (coordinator) and Pico 2 (each leaf) — download from micropython.org
  • A file upload tool: Thonny IDE (beginner-friendly) or mpremote via pip install mpremote
  • This repo's firmware/micropython/src/ folder and web/app/ folder cloned locally
🌐 Coordinator Pico 2 W (has WiFi)
1
Build coordinator config
Open Config Builder, select role Coordinator. Fill in your WiFi credentials, unit name, and the IDs of your leaf units. Download config.json.
2
Generate sun times if needed
Only required if any LED schedule uses sunrise or sunset keywords. The Sun Times section appears automatically inside Config Builder — set your location and download sun_times.json.
3
Flash MicroPython
Hold BOOTSEL on the Pico 2 W while plugging in USB. It mounts as a drive. Drag the Pico 2 W UF2 onto it. The board reboots into MicroPython.
4
Upload firmware files
Using Thonny or mpremote, copy the entire src/ folder to the board root. Then copy web/app/ to a folder named /www on the board — this is the web dashboard served over WiFi.
5
Upload config
Copy config.json (and sun_times.json if generated) to the board's filesystem root — same level as the src/ folder.
6
Power on
Unplug and reconnect power. The coordinator connects to your WiFi and starts the web dashboard. Find its IP on your router, or try lokki-<unitname>.local.
🍃 Each Leaf Unit Pico 2 (no WiFi)
1
Build leaf config
Open Config Builder, select role Leaf. Set a unique unit ID (1–8), configure LED channels, schedules, and any PIR sensors. Download config.json.
2
Flash MicroPython
Hold BOOTSEL on the Pico 2 while plugging in USB. Drag the Pico 2 UF2 onto the mounted drive. The board reboots into MicroPython.
3
Upload firmware & config
Copy the src/ folder to the board root, then copy config.json alongside it. Leaves do not need the /www folder.
4
Power on
Plug in power. The leaf starts up and listens for the coordinator over LoRa. No WiFi needed — it syncs automatically once the coordinator is online.
✅
Verify everything is working
Open the coordinator's web dashboard in a browser. The Fleet table should show all leaf units as Online within a minute or two of powering them on. If a leaf shows Offline, check that its unit ID matches what the coordinator has in its Leaf Unit IDs config, and that all units are on the same LoRa channel and frequency.
